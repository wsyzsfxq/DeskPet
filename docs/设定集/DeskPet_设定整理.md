# DeskPet 设定整理

本文档用于沉淀 DeskPet 当前阶段的项目设定、产品方向和世界观想法。中文正式名暂定，`DeskPet` 作为英文项目代号使用。

## 一句话定位

DeskPet 是一个桌面常驻宠物与轻量 AI 陪伴项目。它不住在传统游戏窗口里，而是住在用户真实的电脑桌面上。

## 当前核心理解

DeskPet 不是一个需要用户专门打开大窗口进入的宠物养成游戏。

它的主要特色是：平时可以隐藏宠物的窝、家具、状态面板、喂养界面、设置界面等 UI，只在桌面上保留一只正在活动的宠物。用户想要照顾、装扮、配置或查看状态时，可以一键把隐藏的 UI 重新放出来。

因此，DeskPet 的第一体验应该是：

> 你的桌面上真的住着一只小东西。

它可以在桌面上活动、待机、睡觉、看用户操作、对鼠标点击作出反应，也可以回到自己的窝里和家具互动。后期在用户明确开启权限后，它可以看见或理解桌面上的内容，知道主人正在做什么，并以宠物行为、短句、提醒或助手能力进行回应。

## 体验关键词

- 桌面常驻
- 低打扰陪伴
- UI 可隐藏
- 一键唤出完整界面
- 宠物与鼠标直接互动
- 宠物与窝、家具、装扮互动
- 可选的桌面感知能力
- AI 陪伴，而不是单纯聊天机器人
- 温柔、安静、稍微有点奇异

## 产品形态

### 桌面上的宠物

宠物应该能够独立存在于桌面上，而不是被困在一个明显的游戏窗口里。

基础行为包括：

- 在桌面上移动、停留、发呆、睡觉。
- 对鼠标点击、悬停、拖拽、重复打扰等行为作出反应。
- 在用户工作、写作、画图、看视频、玩游戏时保持低打扰陪伴。
- 支持快速隐藏、显示、置顶或切换互动模式。

### 可隐藏的窝和 UI

宠物的窝以及各种 UI 界面都可以隐藏。隐藏后桌面只留下宠物本身。

可隐藏内容包括：

- 宠物窝或房间。
- 家具与装扮编辑。
- 喂养、清洁、互动、状态等养成界面。
- 设置、权限、桌面感知开关。
- 背包、商店、收藏等扩展系统。

用户需要时，可以通过一个明确动作把这些界面展开。

### 窝和家具互动

宠物的窝不是静态背景，而是宠物真正会使用的生活空间。

可能的互动：

- 在床、窝、纸箱、垫子里睡觉。
- 在家具上爬、躲、趴、发呆或玩耍。
- 对新家具表现出好奇、喜欢、不喜欢或无视。
- 不同家具影响宠物的待机动画、情绪、作息或互动选项。
- 家具和装扮共同形成每个玩家独有的桌面生活感。

### 鼠标点击互动

用户可以不打开完整 UI，直接用鼠标和宠物互动。

可能的互动：

- 点击宠物，让它回头、眨眼、叫一声或靠近。
- 长按或连续点击触发不同反应。
- 拖动玩具或小物件让宠物追逐。
- 鼠标靠近时，宠物观察、躲避、追随或扑向光标。
- 宠物对过度打扰表现出不满，对温和互动表现出亲近。

## AI 与桌面感知方向

后期希望宠物能够在用户允许的情况下“看见”桌面上的东西，知道主人正在做什么。

这部分必须有清晰开关，并且默认应尊重隐私。

可能能力：

- 判断用户正在写文档、画图、编码、开会、看视频或玩游戏。
- 识别长时间工作、深夜使用、频繁切换窗口、长时间空闲等状态。
- 通过动作陪伴，比如趴在窗口边、看用户工作、送来虚拟小物件。
- 通过短句提醒，比如休息、喝水、别忘记保存、会议要开始等。
- 在需要时发展成轻量助手，但不抢走宠物本身的角色感。

设计原则：

> AI 能力应该让宠物更像一个有感知的陪伴者，而不是把它变成一个套着宠物皮的聊天框。

## 世界观方向

早期设定里有“余生物”的概念：现代城市或日常缝隙中，有一些没有被记录、没有被看见的小生命。它们并不承担拯救世界的使命，只是安静地存在，等待被注意到。

这个概念仍然可以作为 DeskPet 的情绪底色。

但在当前产品形态下，世界观需要从“城市缝隙”转向“数字桌面缝隙”：

- 它不是住在游戏房间里，而是住在桌面边角。
- 它不是被摄像头拍不到的城市生物，而是只在你的电脑环境中显形的小生命。
- 它可以和现实软件窗口、鼠标、文件、时间、天气、用户习惯发生关系。

可以保留的概念句：

> 它不住在游戏窗口里，它住在你的桌面上。

## 情绪气质

DeskPet 的气质不是强打扰、强效率、强工具化，而是低打扰的陪伴。

它可以可爱，但不只是卖萌。

它可以智能，但不应该像工作软件一样催促用户。

它可以有荒诞感，比如奇怪的食物、隐藏的小市场、突然出现的物件、桌面边缘的小秘密，但这些内容应该服务于“它真的在这里生活”的感觉。

目标情绪：

- 温暖
- 安静
- 陪伴
- 好奇
- 稍微有点神秘
- 让用户觉得被看见，但不被监视

## 养成方向

原始设定中的养成系统仍然可作为长期设计储备：

- 饱食度、营养、清洁、心情、健康等隐性状态。
- 状态不直接用数值条暴露，而通过动作、外观、声音、行为表现。
- 喂食、清洁、互动、装扮会影响宠物的状态和习惯。
- 长期照料形成历史，不应像普通数值一样随时无痕重置。
- 宠物可以生病、落寞、变胖、变瘦、慢慢养成个人习惯。

需要注意的是，桌面宠物版本不应把养成压力做得太重。核心应是陪伴和关系，而不是日常任务负担。

## 当前优先级

第一阶段优先证明桌面宠物形态：

- 透明或异形桌面窗口。
- 宠物常驻桌面。
- Always-on-top 行为。
- 鼠标点击互动。
- UI 隐藏与一键展开。
- 简单窝界面。
- 基础待机、移动、睡觉等动画状态。

第二阶段再扩展养成与生活感：

- 家具互动。
- 喂养、清洁、心情等基础循环。
- 宠物离线或待机行为。
- 桌面环境中的小事件。
- 更丰富的点击和鼠标互动。

第三阶段探索 AI 与桌面感知：

- 明确的桌面感知权限开关。
- 局部、可解释、可关闭的桌面理解。
- AI 行为驱动的宠物反应。
- 轻量助手能力。
- 长期记忆和关系感。

## 待定问题

- 中文正式名。
- 首只宠物的视觉风格。
- 使用 2D、Live2D、Spine、像素动画、3D，还是混合方案。
- 技术栈选择：Tauri、Electron、Unity、Godot、原生 Windows 等。
- 窗口穿透、点击捕获和桌面置顶的具体实现。
- AI 能力是本地优先、云端模型，还是混合方案。
- 隐私与权限说明如何设计得足够清楚。
